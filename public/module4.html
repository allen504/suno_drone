Module 4: Arduino & Control Code

In this module, students bring the drone to life with embedded code. They set up the programming environment, read sensor data, drive motors, and perform controlled tests before attempting a full flight.

Main curriculum weeks: 21 (FPV Camera), 22 (Programming Setup & Initial Code), 23 (Sensor Calibration), 24 (Motor Testing), 25 (Bench Flight & Safety Check)

Week 21 – FPV Camera

Students install and wire an FPV (First-Person View) camera system, learning how video integrates with the control system for piloting.

Goals

  • Understand the basic components of an FPV system.
  • Wire power and signal for the FPV camera and transmitter.
  • Mount the camera securely on the drone frame.

In-Class Activities

  • Inspect an FPV camera and video transmitter (VTX) setup.
  • Wire the system to the drone’s power distribution safely.
  • Test the video feed and discuss latency and signal quality.

Why This Matters

FPV is a common way to fly drones. Understanding how video and control systems interact prepares students for real-world piloting and troubleshooting.

Week 22 – Programming Setup & Initial Code

Students set up their Arduino programming environment and load starter firmware that reads sensors and prepares to drive motors.

Goals

  • Install and configure the Arduino toolchain.
  • Connect the board (e.g. via FTDI) and upload sample code.
  • Understand the structure of the flight-control firmware.

In-Class Activities

  • Walk through installing the IDE or code editor.
  • Upload a simple “blink” or serial-print program to verify the connection.
  • Load the base flight-control sketch and identify key sections of code.

Why This Matters

This week establishes the software pipeline. If students can reliably program the board, every later code change becomes much more manageable.

Week 23 – Sensor Calibration

Students calibrate the IMU (e.g., MPU6050) and verify that gyro and accelerometer readings are stable and meaningful.

Goals

  • Understand why sensors require calibration.
  • Calibrate offsets and scaling for the IMU.
  • Check sensor readings for noise and drift.

In-Class Activities

  • Run calibration routines from the flight firmware or helper sketches.
  • Visualize sensor data over time (serial monitor, plots, etc.).
  • Record calibration values and update configuration constants as needed.

Why This Matters

A drone can’t stabilize itself without trustworthy sensor data. Calibration connects abstract math to precise, real-world performance.

Week 24 – Motor Testing

With drivers wired and sensors calibrated, students carefully test motor spinning, direction, and responsiveness under code control.

Goals

  • Verify that each motor spins when commanded.
  • Check that motor directions match the intended configuration.
  • Observe how code changes affect motor behavior.

In-Class Activities

  • Run test routines that spin motors one at a time.
  • Swap leads or update code to fix reversed motors.
  • Discuss throttle curves, minimum spin thresholds, and safety precautions.

Why This Matters

Motor tests catch issues before the drone ever leaves the ground, preventing crashes due to swapped wires or buggy code.

Week 25 – Bench Flight & Safety Check

Students perform “bench flight” tests, tethering or restraining the drone while running full control code to verify stability and safety.

Goals

  • Run the full control loop with the drone safely restrained.
  • Check for abnormal vibrations, oscillations, or runaway behavior.
  • Confirm that emergency stop procedures work.

In-Class Activities

  • Secure the drone to a test rig or bench.
  • Arm and throttle up slowly while monitoring behavior.
  • Practice disarming quickly in response to unsafe conditions.

Why This Matters

Bench testing teaches structured, safety-first debugging before taking the risk of actual free flight.
